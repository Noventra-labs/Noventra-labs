## What is Noventra Labs?

Noventra Labs is a GitHub organization built around one idea: **most teams build what they imagine is needed — we build what data proves is missing.**

We are not a startup. We are not a VC-backed venture. We are a collective of researchers, engineers, and product thinkers who scan markets, validate gaps, and ship open-source tools to fill them. Every repository in this org exists because a real, recurring, underserved need was found — not assumed.

> *"The best products aren't invented — they're discovered in the gap between what exists and what's genuinely needed."*

---

## How we work

We follow a four-step process from signal to shipped product:

```
SCAN  ──►  VALIDATE  ──►  BUILD  ──►  ITERATE
```

| Phase | What happens |
|---|---|
| `01 · SCAN` | Mine Reddit, GitHub Issues, ProductHunt, forums — surface recurring unmet needs with quantifiable frustration |
| `02 · VALIDATE` | Before a single line of code: surveys, landing page tests, community polls — we measure demand, we don't guess |
| `03 · BUILD` | Repos go public from day one. We build fast, document decisions openly, and iterate with community feedback |
| `04 · ITERATE` | Usage data and user feedback drive the roadmap — not internal assumptions or stakeholder wishes |

## Our principles

**Evidence over intuition.**  
We trust data. Every assumption gets tested before it becomes a build decision. The word "obvious" is not in our vocabulary.

**Radical openness.**  
Research, code, and decisions — all public. We build trust by building in the open. No private roadmaps. No black boxes.

**Shipping over planning.**  
A working prototype beats a perfect roadmap. We move fast, document why, and adjust based on what we learn.

**Community as co-founder.**  
The people who use what we build shape where we go next. Issues, feedback, and PRs aren't noise — they're the signal.

## Contributing

We welcome contributors who prefer evidence over assumptions and shipping over theorizing. Here's how to get involved:

**Found a gap worth investigating?**
Open a [Gap Proposal](https://github.com/noventralabs/meta/issues/new?template=gap-proposal.md) — include the problem, where you've seen it, and rough signal data (upvotes, complaint threads, etc.).

**Want to build with us?**
Check the `good first issue` and `help wanted` labels across our repos. All work is discussed openly in issues before any code is written.

**Want to research with us?**
Every gap investigation has a public research thread. Add sources, data, or counter-evidence — all of it helps us validate faster.

```bash
# Get started
git clone https://github.com/noventralabs/<repo>
cd <repo>
# See CONTRIBUTING.md inside each repo for setup instructions
```

## Design system

All Noventra Labs products follow the **Precision Dark** design language:

- `#0a0a0b` dark base · `#e8a237` amber signal · `DM Mono` for all data
- Sharp edges, no border-radius · borders over shadows · amber means *found it*
- The 2×2 gap-mark grid appears in every product as a visual fingerprint

Full spec → [`meta/design-system/`](https://github.com/noventralabs/meta)
